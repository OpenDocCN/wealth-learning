# 6.反馈一点后台数据分

反馈一点后台数据分析的功能：

我们都知道微信公众号和小程序的后台是提供数据分析功能 的。那些功能可以满足大部分分析需求了，小程序甚至提供 了自定义数据分析的功能，开发者可以埋点获取更多的分 析。

但是这依然不太够，或者很多号主、开发者并不知道这些数 据分析功能（所以新媒体数据分析师们有饭吃）。

这就涉及到一个后台数据分析功能的设置。

微信公众号的还算好使，但是知识星球后台数据分析功能就 差点意思了。

【先来段吐槽】

说实话，知识星球的这个数据分析功能从我使用的体验来看 几乎提供不了什么有价值的信息。

功能入口在网页版—个人信息—数据分析。进入功能你会看 到一个收入数据、用户活跃数据、又一个收入数据（赞 赏）、内容生产数据，还有通过星主分享的二维码进入的用 户数据。

看上去维度非常全，但实际上没啥能用的。

第三个图我是通过手工将活跃人数（第一个图）和星球主题 数（第二个图）的数据记录到 Excel 得出的，因为这两个图表

的“下载数据”导出的数据乱到一脸懵逼（图四）：导出的是 以星球用户为“主语”的数据。

另外，这俩图表中的数据不是全部的数据，有些日期的数据 是缺失（看图表应该是为了维护图表的连续性，那些缺失的 数据都是没有产能的 0）。

然后可以看第五个截图里面二维码分析，最后两个时间是 1970 年和公元 80 年 因为这个错误数据，我只能直接放弃对这个后台数据的信任 了。

【功能设置】

如果我没理解错，添加数据分析功能应该是帮助星主们提供 一些运营支撑的，或者最起码用来记录和回忆。

首先不谈功能，最起码应该保证数据的有效性，明显的错误 和遗漏不应该存在。

然后常用的数据，如果后台不能提供交叉分析应该提供导出 功能。图表都有了，导出数据肯定是可以的。当然，这里的 导出应该和图表一一对应，图表是什么导出的就应该是什 么。

最重要的，提供哪些分析功能。 我认为两个维度的数据是必不可少的：行为数据和传播数据

（效果数据）。

有这两个维度的数据，星主才能用来对比“付出”和“收获”之间 的关系。

「行为数据」也就是星主主动去做的事情，可以是分享二维 码、发布主题、组织活动、置顶主题等。

「传播/效果数据」星主主动行为的结果，可以是活跃度、阅 读数、付费人数等。

【期待更好】

知识星球功能越来越丰富，提供的可见数据越来越多，这对 于星球运营者来说是一件非常利好的事情。

大部分星主都是有一定运营能力的大 V，都有很好的数据驱动 运营思维和能力，这些能力的优化可以帮助他们更好的打磨 星球的运营。

当然，数据分析是一件“锦上添花”的事情，可能在开发中优 先级不高。

但值得期待。
