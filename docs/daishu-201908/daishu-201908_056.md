# 前段时间休假在家没

奔跑 : 前段时间休假在家没电脑，回到单位马上抽时间体验了 WCplus2.0，真的是神器，非常实用，由衷的佩服阿呆哥。目 前一共采集了 6 个公众号，在使用中也总结了一些问题和建 议，因为本人是开发小白，单纯从用户角度考虑的，说的不 对的地方请批评指正，请轻喷...

存在问题: 1.电脑断网重连后会从头开始；

2.C 区的导出操作需要刷新页面才出来；

3.全部采集完后，导出 PDF 时只导出了一小部分，很多文章提 示“尚未采集文章正文 无法到处到 PDF 为了得到全部文章的 PDF 建议重新采集一次”，重新采集后还是不行，后来无意中 又点了一次导出，居然可以了。

拙见:

1.采集过程中断网重连会从头开始，能否提供暂停、继续功 能？有事时能够手动暂停，断网等情况自己暂停；

2.PDF 导出的文件数量与文章总数不一致，能否清晰的告知用 户少了哪几篇？便于用户整理和补充；

3.若定位为面向广大用户的产品，尝试提高用户体验，A 区不 需要知道获取每一类参数已过时分秒，只需告诉用户采集已

经开始了多长时间，预计还有多长时间结束，D 区采集时公众 号栏下的尚需分秒也可去掉，统一在一个地方显示预计多久 结束；

4.能否采集到留言信息，很多文章下面的留言也非常有价值， 能否加到 PDF 的正文后面；

5.提供任务列表，列出多个待采集的公众号，可选择采集完自 动导出 Excel、PDF，甚至可以任务全部完成后自动关机等操 作。
