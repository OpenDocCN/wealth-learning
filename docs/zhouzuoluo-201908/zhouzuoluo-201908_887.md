1.

你不知道的东西

用户更加不知道

你是不是经常需要写产品类型的文案？相信以下这种情形， 你经常会遇到。

比如客户扔一本说明书给你，让你自己找卖点，那些卖点包 括以下这些数据，你会怎么找？

这个是楼盘的数据： 占地面积：46084 平方米；建筑面积：358000 平方米；容积

率：5.90；绿化率：30%；停车位：1570 个等。

这个是手机的参数：

屏幕大小：5.0 英寸；像素密度：320ppi；电池容量： 2500mAh；摄像头：5300 万像素；重量：160g ；视频拍摄：3 倍变速等。

或者有时候你需要进一步了解产品，和设计师、工程师聊了 大半天，你却听得一脸懵逼，因为从他们口中说的，都是专 业术语：

这是说钻石行业的术语：

刻面、素面、蛋面、有烧 、无烧、山料、籽料、A 货、B 货、

C 货爪镶、包镶、夹镶、大三、小四、中五……

这是说化妆品行业的术语：

SKU、SPF、PA、BB 霜、CC 霜连带率、增长率、贴花……

这个时候，你遇到数据化的卖点、或者是专业化的术语，如 何办呢，是直接将数据摆上去吗？当然不是。

因为连你都不知道的东西，用户更加不知道。 你的产品数据可能非常好，但是再好都是产品的事情，跟用

户没关系。

用户不关心你数据、不关心你的术语，用户只关心自己，而 只有你的数据或者术语让其能立即认知，他们才会关心。

那么，有什么好方法可以解决呢？

2.

将你不熟悉的事情 转换为熟悉的事物

这里有一个非常简单的方法，就是将你的数据、术语转化为 熟悉的事物。

用户对你数据没有感觉，是因为他不知道你说的具体将会是 什么，不清楚这个数据从何而来，也不清楚为什么数据如此

厉害等等。

那我们就将数据或术语翻译一遍，将其翻译成用户的大脑能 迅速认知的熟悉的事情。用户能读懂熟悉的事物，自然就读 懂你的数据或者术语。

为什么会这样呢？因为人的大脑更加容易理解和记忆视觉化 的信息，当看到一个陌生的事物，会马上抽取过去再大脑存 在的记忆信息，转换成自己喜欢的视觉化信息，而不是抽象 的信息。

3.

用已知的事物 解释未知事物

那么，我们具体如何运用呢？ 再用前文手机参数作例子。很多人买手机，都第一时间打开

手机摄像头来拍几张照片，在这么短的时间内，他也只能做

这一步动作。

如果你和他说，我的像素密度：320ppi；电池容量： 2500mAh；摄像头：5300 万像素……用户能记得住那么多 么？当然是不可能的！

努比亚是这样告诉用户的“这一台可以拍星星的手机”，即使 连星星都这么远都可以拍得到，是不是很厉害？

用户第一时间就记得了你的特点，而不是 5300 万像素。 所以，以后大家写文案的时候，数据如果一定要说，那可以

试试将数据嫁接到用户熟知的事物上，借用已知的事物来解 释未知事物。
