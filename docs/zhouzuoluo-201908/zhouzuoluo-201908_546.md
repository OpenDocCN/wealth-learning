# 《永远不要降低自己

《永远不要降低自己的工作标准，包括合作伙伴》

因为一张图片设计，粥老师吐槽我的设计师工作的底线低， 原因是设计师做的图片跟我们的竞品太像了。

我起初觉得这个说法太武断了，毕竟设计师已经反反复复改 图，一直没达到我的标准，最后没办法了，她照着我给的模 版做了几个方案。

后来我反思：难道因为已经改了很多次，就可以直接 copy 人 家的图吗？明明没有达到我心目中的标准，难道因为已经改 了很多次，我就可以降低标准勉强接受吗？

从感情上，我可以体谅设计师的不容易，但站在工作的角 度，谁也不会得到额外的宽容。

就比如说我们写一篇文章，可能绞尽脑汁把最好的水平都拿 出来了，熬着夜反复修改。但如果文章质量不行，你再辛苦 也不会有阅读量；就比如我们的图片，如果对竞品公司看 到，只会觉得我们抄袭没品。至于背后的原因，真的没那么 重要。

梁宁曾在《产品思维 30 讲》提到，很少的女性能做好产品经 理，我还有点不服，现在我发现真的不容易做到。

卓越的产品经理一定是非常严苛和自我。比如说她的一个同 事，曾经让设计师修改了一百多次，就是为了一个开机不到 0.1 秒的界面。

结果搞到设计师要辞职，梁宁跟他说：“设计师被你搞疯了， 要走了。”

他呢，直接回答：“哦！那新的设计师什么时候来，我还得改 啊。”

梁宁诧异对方居然毫不关心设计师的感受，只关心是不是已 经完美了，听起来好自私哦。可是如果没有追求极致的精 神，怎么能打造一款极致的产品？

回到其他工作，如果不是严苛要求，又怎么可能把工作做完 美？回到自身，如果不是处处对自己高要求高标准，怎么会 快速进步，又怎么能出类拔萃呢？

有的时候，我们可能会不自觉地放低对自己标准，比如“我今 天太累了，差不多得了”、“别人都是这样的，我这已经不错 了”；有的时候因为外界的原因“你太挑剔了，差不多了”、“你 的要求我做不到，太严苛了”。
